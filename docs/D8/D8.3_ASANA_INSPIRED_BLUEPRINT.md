# D8.3 — Asana-inspired UX blueprint (FREEDOLIAPP, Amazon-first)

Contracte UX per mantenir coherència i que no es perdi el disseny adoptat.

---

## 1. Principis adoptats

- **Jerarquia clara**: una decisió principal per pantalla; títol + subtítol + cos + CTA.
- **Progrés visible**: stepper / pipeline visible (wizard, dashboard Mode B).
- **Estats humans**: loading, empty, error amb missatges clars; no pantalles en blanc.
- **8px grid**: spacing consistent (tokens `--s-1` … `--s-6`).
- **Responsive**: mobile-first; columnes que col·lapsen; stepper amb dots a mòbil.
- **Amazon-first**: el flux principal és Connect Amazon → Import → Snapshot → Dashboard; Setup mode com a alternativa sense dades.

---

## 2. Principis NO adoptats

- **Onboarding task-based tipus Asana**: no llistes de tasques d’onboarding; en el nostre cas el wizard (Connect → Import → Snapshot → More tools) és el flux únic d’“onboarding” amb dades.
- **Workspaces múltiples per usuari** com a primer citizen: prioritzem una org activa; canvi d’org via selector.

---

## 3. Mapping de tokens (`tokens.css`)

| Propòsit        | Tokens |
|-----------------|--------|
| Spacing         | `--s-1` … `--s-6` |
| Radius          | `--r-sm`, `--r-md`, `--r-lg` |
| Shadows         | `--shadow-sm`, `--shadow-md` |
| Typography      | `--text-1`, `--text-2`; mides via components |
| Semantic colors| `--color-primary`, `--color-danger`, `--color-warning`, `--color-success` |
| Surface/BG      | `--bg`, `--surface`, `--border` |
| Buttons         | `--btn-h-sm/md/lg`, `--btn-px-sm/md/lg`, `--btn-font` |

Tots els components UI han d’usar aquests tokens; no colors hex directes.

---

## 4. Regles UI

- **Zero hardcode**: no `style={{}}`, no hex, no textos literals en botons/headers.
- **Botons**: només via component canònic `Button` (`src/components/ui/Button.jsx`).
- **i18n obligatori**: tots els textos visibles via `useT()` / `t()` i claus a `locales/*.json`.

---

## 5. Pantalles i patrons

### Landing

- Hero + 2 CTAs (Login / Sign up o equivalent).
- Proof cards (testimonials / logos).
- Footer mínim.

### Login

- 1 columna, inputs nets (Input canònic).
- Errors humans (missatges traduïts).

### Wizard (activation)

- Connect Amazon → Import → Snapshot → (opcional) More tools → Dashboard.
- Stepper visible; una decisió per pas; CTAs “Continue” / “Skip for now” / “Enter Dashboard”.

### Dashboard

- **Dual-mode**:
  - **Mode A (amb dades)**: analytics, KPIs, Risk Radar, Money Focus, widgets (orders, POs, finances, etc.).
  - **Mode B (sense dades)**: “Getting started” amb pipeline (Create product → Viability → Quotes → Samples → PO → Shipment → Amazon).
- **Criteri de detecció**: presència d’activitat recent al `financial_ledger` (org, scope company, últims 30 dies) o almenys 1 fila; si no hi ha dades o error → Mode B.
