# D8.3.1 — UI Foundation (tokens + canonical components + i18n + lint)

## 1. Tokens únics
- **Fitxer**: `src/styles/tokens.css`
- S'hi defineixen els tokens canònics:
  - Colors semàntics: `--color-primary`, `--color-danger`, `--color-warning`, `--color-success`
  - Text: `--text-1`, `--text-2`
  - Surface/background: `--bg`, `--surface`, `--border`
  - Radius: `--r-sm`, `--r-md`, `--r-lg`
  - Shadow: `--shadow-sm`, `--shadow-md`
  - Spacing: `--s-1`..`--s-6`
  - Botons: `--btn-h-sm/md/lg`, `--btn-px-sm/md/lg`, `--btn-font`
- **Import global**: `src/main.jsx` importa `./styles/tokens.css` abans de `index.css`.

## 2. Components canònics (UI)
- **Carpeta**: `src/components/ui/`
- Components:
  - `Button.jsx`
    - API: `variant` (`primary|secondary|danger|ghost`), `size` (`sm|md|lg`), `disabled`, `loading`, `fullWidth`, `className`.
    - No accepta `style` (s'ignora si es passa); estils només via classes i tokens (`.ui-btn*` a `src/styles/ui.css`).
  - `Input.jsx`
    - Props: `label`, `hint`, `error`, `className`, `...inputProps`.
    - Usa classes `ui-field`, `ui-input*`.
  - `Select.jsx`
    - Mateix patró que `Input`, amb `ui-select*`.
  - `Card.jsx`
    - Card base (`ui-card`) amb variant `elevated` (`ui-card--elevated`).
  - `Badge.jsx`
    - Badges amb variants `neutral|success|warning|danger|info` (classes `ui-badge*`).
- **CSS UI únic**: `src/styles/ui.css`
  - Defineix totes les classes anteriors i només utilitza tokens (no colors hex directes).

## 3. I18n únic
- Sistema canònic: **i18next + react-i18next**
  - Inicialització: `src/i18n/index.js`
  - Traduccions: `src/i18n/locales/en.json`, `ca.json`, `es.json`
- S'ha afegit:
  - `common.buttons.save/cancel/continue/upgrade`
  - `billing.banner.trialEnded/pastDue/upgradeCta`
- Helper:
  - `src/hooks/useT.js` retorna `t` de `useTranslation()` per evitar imports repetits.
- Sistema antic de `messages.js` / `t.js` marcat com **DEPRECATED** (comentari a dalt del fitxer); només s'utilitza per compatibilitat en views existents (`BillingLocked`, `BillingOverSeat`). Codi nou ha d'usar i18next.

## 4. Regles “no hardcode”
- Nou codi UI ha de:
  - Fer servir `Button`/`Input`/`Select`/`Card`/`Badge` de `src/components/ui/`.
  - No afegir `style={{ ... }}` en components nous.
  - No introduir colors hex directes a JSX/TSX (fer servir tokens).
  - Text visible nou ha de passar per `t()` (`useT()` o `useTranslation()`).
- Exemple d'aplicació:
  - `BillingBanner` fa servir ara `Button` canònic + i18n (`billing.banner.*`) en lloc de text hardcoded + inline styles.

## 5. ui:lint (enforcement lleuger)
- Script: `scripts/ui-lint.js`
- Com s'executa:
  - `npm run ui:lint`
- Comportament:
  - Només analitza **fitxers staged dins `src/`** (`git diff --cached`).
  - Falla si troba:
    - `style={{` (inline styles nous).
    - `#RRGGBB` o variants (colors hex).
    - `<button>Text</button>` amb text literal sense `t(` (heurística per evitar botons sense i18n ni component canònic).
