# D8.2 — Billing & Plan Enforcement (ORG-level) — Stripe (1 producte + N prices)

## 1. Objectiu
Fer FREEDOLIAPP **monetitzable** amb billing org-level:
- Trial **14 dies** sense targeta
- Plans: `growth`, `pro`, `agency`
- Enforcement per límits: **seats** i **Amazon connections**
- Core app no es bloqueja (només features/límits)

## 2. Decisions tancades
- Stripe model: **1 producte** + **3 prices** (growth/pro/agency)
- Trial comença **després** de completar onboarding (`org_activation`)
- Billing és **per org**, no per user ni per brand

## 3. Model d'estat (state machine)
Estats mínims:
- `trialing`
- `active`
- `past_due`
- `canceled`

Regles:
- `trialing` fins `trial_ends_at`
- upgrade → `active`
- impagament → `past_due`
- cancel·lació → `canceled`

## 4. Plans i límits (canònic v1)
### Seats
- growth: 1
- pro: 5
- agency: 15

### Amazon connections (SP-API)
- growth: 1
- pro: 3
- agency: 10

### Feature gates v1
- Export packs: pro+
- Advanced alerts: pro+
- Priority ingest: agency

## 5. Taules (DB contract)
### `org_billing` (1 fila per org)
Camps:
- `org_id` (PK, FK organizations)
- `stripe_customer_id`
- `stripe_subscription_id`
- `plan` (growth|pro|agency)
- `status` (trialing|active|past_due|canceled)
- `trial_started_at`
- `trial_ends_at`
- `current_period_end_at` (opcional però útil)
- `created_at`, `updated_at`

Notes:
- `org_billing` és la font de veritat per gates.
- No barrejar-ho amb `org_activation`.

## 6. Activació del trial (B)
Trigger lògic:
- Quan existeix `org_activation.activation_completed_at` per org
→ crear `org_billing` si no existeix:
- `status='trialing'`
- `plan='growth'` (default)
- `trial_started_at=now()`
- `trial_ends_at=now()+14 days`

## 7. Stripe flow (alt nivell)
- Settings → "Upgrade"
- Stripe Checkout (price segons pla)
- Webhook Stripe → actualitza `org_billing` (customer/subscription/status/plan/period_end)

## 8. Enforcement (product rule)
Quan `past_due` o trial expirat:
- Mostrar banner persistent (BillingBanner)
- Bloquejar només:
  - afegir seats nous (quan supera límit)
  - crear noves SP-API connections (quan supera límit)
  - features premium (exports/advanced alerts)
- No bloquejar lectura ni operacions core ja existents.

## 9. Observabilitat mínima (link amb F6)
Registrar a `ops_events`:
- billing.trial_started
- billing.checkout_started
- billing.subscription_updated
- billing.payment_failed (si arriba del webhook)
(No implementar analytics ara, només contracte.)
