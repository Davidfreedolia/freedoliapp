# D8.1 — Activation Wizard (No-skip) — Amazon optional

## 1. Objectiu de la fase
Fer que qualsevol org nova quedi **activada** amb fricció mínima i sense bloquejar usuaris sense vendes:
- Onboarding obligatori (no hi ha skip global).
- Elecció obligatòria de camí: **Amazon** o **Setup**.
- Amazon és opcional.
- El wizard només passa una vegada.

## 2. Principis de disseny
- **No skip global**: tota org nova entra al wizard.
- **Camí obligatori**: l’usuari tria Amazon o Setup.
- **No dependre de vendes**: zero activitat no pot bloquejar.
- **Una sola font de veritat**: `org_activation`.
- **Wizard no reapareix** un cop completat.

## 3. Model de dades
Taula mínima per activació (única font de veritat):

```sql
-- docs only (migració a la fase d'implementació)
create table if not exists org_activation (
  org_id uuid primary key references orgs(id) on delete cascade,
  activation_completed_at timestamptz not null,
  activation_path text not null check (activation_path in ('amazon','setup'))
);
```

Notes:

- No és un boolean: necessitem traçabilitat del camí.
- No depèn de ledger / events: Amazon és opcional i pot no haver-hi dades.
- Serveix per mètriques i segmentació futura.

## 4. Routing i guard (hard gate)

Definició:

`requiresOnboarding = NOT EXISTS (org_activation for org_id)`

Ruta dedicada: `/activation` (fora del layout principal).

Regles:

- Si `requiresOnboarding=true` i no estàs a `/activation` → redirect a `/activation`.
- Si `requiresOnboarding=false` i estàs a `/activation` → redirect a `/app`.
- Logout permès durant el wizard.

## 5. Flux del wizard (pas a pas)

**Step 1 — Confirm Org**

- Nom org
- Base currency
- Marketplace principal (si aplica)
→ Continue

**Step 2 — Choose Path (obligatori)**

Opcions:

- Connect Amazon & import real data
- Start in Setup Mode (no Amazon yet)

No hi ha botó “Skip”. Cal triar una opció.

## 6. Camí AMAZON

**Step 3 — OAuth SP-API**

- Connexió robusta (ja existent).
- Validació connexió OK abans de continuar.

**Step 4 — First ingest (auto)**

- Rang fix: last 30 days
- Trigger “on demand” (no esperar cron) per millorar UX.

**Step 5 — First Snapshot**

Mostrar (minimal):

- Revenue
- Amazon Fees
- Ads
- Net Profit
- Cash Impact

Zero activity mode:

- Si el rang retorna 0 dades: mostrar missatge clar (“No activity detected in last 30 days…”) i permetre entrar igualment.

En clicar “Enter Dashboard”:

- Escriure `org_activation` amb `activation_path='amazon'` i `activation_completed_at=now()`.

## 7. Camí SETUP (sense Amazon)

**Step 3 — Setup Mode Activated**

Missatge:

- Pots crear productes/projectes, planificar costos i preparar el launch.
- Pots connectar Amazon més endavant a Settings.

En clicar “Enter Dashboard”:

- Escriure `org_activation` amb `activation_path='setup'` i `activation_completed_at=now()`.

## 8. Edge cases

- Org nova sense vendes → no bloquejar (Setup o Zero activity).
- OAuth fallit → retry i missatge clar.
- Ingest triga (Amazon report generation) → estat informatiu, no spinner infinit.
- Usuari fa logout → en tornar, continua wizard (mentre org no estigui activada).
- Eliminació d’org → cascada de `org_activation`.

## 9. Mètriques (previst per FASE 8.4)

Amb `org_activation` podem mesurar:

- % Amazon vs Setup
- `time_to_activation`
- `time_to_amazon_connect` (si es deriva d’events/ops més endavant)

(No implementar analytics ara.)

## 10. Implementation Notes (as-built)

- **On-demand ingest**: `spapi-settlement-worker` accepta POST amb `{ connection_id }` i executa ingest només per aquesta connexió (sense lock global).
- **Wizard Amazon path**:
  - Step Connect: OAuth init/callback existents; detecta connexió activa i mostra "Connected ✓".
  - Step Import: invoke Edge `spapi-settlement-worker` POST; polling `amazon_import_jobs` cada 4s fins `done` o 2 min.
  - Step Snapshot: calcula via `financial_ledger` last 30 days (scope company).
  - Activation: INSERT `org_activation` path='amazon' (dup key ok).
- **OAuth return**: `sessionStorage.activation_amazon_path` + redirect back to `/activation`.

## 11. Deploy checklist (Amazon path)

- `npx supabase functions deploy spapi-settlement-worker`
