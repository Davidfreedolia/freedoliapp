FREEDOLIAPP — PILOT PLAYBOOK V1
================================

1. Pilot status
---------------

- producte **pilot-ready with caveats**
- pilot **controlat i acompanyat**
- **no** self-serve massiu

2. Recommended pilot entry path
-------------------------------

- recomanat: **Setup / OS manual path** (camí d’entrada manual, amb acompanyament)
- **Amazon snapshot path** només per casos preparats i amb suport directe (orgs amb context tècnic i dades Amazon reals)

3. Recommended demo / pilot script
----------------------------------

(*Guió de recorregut per a la primera sessió de pilot / demo guiada.*)

- pas 1: **Landing / context curt**  
  - Explicar en 1–2 minuts què resol FREEDOLIAPP (from project → PO → inventory → profit).
- pas 2: **Activation path triat**  
  - Triar explícitament el camí Setup / OS manual (recomanat) o, en casos molt preparats, Amazon snapshot amb suport.
- pas 3: **Dashboard (first value)**  
  - Ensenyar l’estat org, alertes i tasks mínimes; deixar clar què és “primer valor” per aquell pilot.
- pas 4: **Projects**  
  - Mostrar el catàleg de projectes i com representen operacions reals del pilot.
- pas 5: **ProjectDetail**  
  - Baixar al detall d’un projecte representatiu; explicar com es connecta amb POs, suppliers i inventari.
- pas 6: **Orders**  
  - Enseyar com es creen/gestionen POs i com s’encaixen dins el projecte.
- pas 7: **Inventory només si aporta al cas**  
  - Mostrar inventari quan tingui sentit per al cas pilot (stock, link a projectes, visibilitat de flux).
- finances avançades només si són rellevants per aquell pilot  
  - No centrar el primer pilot en finances avançades; només tocar-les si aporten clarament.

4. Screens to prioritize
------------------------

- **Dashboard**
- **Projects**
- **ProjectDetail**
- **Orders**
- **Suppliers**
- **Inventory** (com a suport quan tingui sentit)

5. Screens to avoid in first pilot unless needed
------------------------------------------------

- parts massa tècniques d’**Activation** (configuració profunda, camins avançats)
- **finances avançades** com a focus principal
- qualsevol **racó legacy** no central al flow Project → PO → Inventory

6. Pilot caveats
----------------

- **onboarding millor guiat**: l’entrada funciona, però el pilot ha de ser conduït amb algú de l’equip
- **Amazon-first més delicat**: reservar-lo a casos preparats i amb suport proper
- **ProjectDetail és potent però dens**: cal narrar-lo i seleccionar què ensenyar
- el pilot s’ha de fer **amb guió** (aquest playbook) i no com a exploració lliure inicial

7. Success criteria for first pilot
-----------------------------------

- l’usuari **entén el flow** Project → PO → Inventory
- l’usuari **veu valor** al **Dashboard + ProjectDetail + Orders**
- no hi ha **bloquejos crítics** de navegació o comprensió durant el recorregut guiat
- es recull **feedback concret**, no només impressions vagues (què ha entès, què falta, què li ha sobrat)

8. Immediate next actions
-------------------------

- triar **1–3 orgs pilot** amb casos prou concrets
- preparar el **path d’entrada** (Setup / OS manual com a default)
- preparar **dades inicials mínimes** si cal (projecte representatiu, POs d’exemple, suppliers bàsics)
- **executar el pilot** amb aquest guió i recollir **feedback estructurat** (notes per bloc i per pantalla prioritzada)
