# FREEDOLIAPP — ENTRY EXPERIENCE

Status: Draft  
Scope: First user experience

---

## Objective

The first experience must deliver value **very quickly**.

A new user should understand:

- what the system does
- what the current situation is
- what actions are needed

---

## Entry Flow


Landing
→ Trial
→ Magic link
→ Activation
→ Home


---

## First Value Moment

When the user opens the application, they should see:

- the current state of the business
- relevant operational signals

This is delivered through the **decision feed**.

---

## Decision-Driven Product

Freedoliapp is structured around the idea that the system helps answer:

- What is happening?
- What is wrong?
- What should I do next?

The interface should surface **decisions**, not raw data.

---

## Key Entry Principle

The system must avoid overwhelming the user with complex analytics.

Instead, the product should highlight:

- important signals
- operational risks
- actionable recommendations
