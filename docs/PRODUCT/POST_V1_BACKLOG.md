# FREEDOLIAPP — POST V1 BACKLOG

Status: Frozen backlog  
Purpose: Capture product ideas that are valuable but **not required to launch V1**.

Rule:
Nothing in this document should delay or interfere with V1 completion.

---

# Product Vision (Long Term)

Freedoliapp aims to become the **operating system for Amazon sellers**, combining:

- operations
- analysis
- decisions
- collaboration

However, V1 focuses only on the minimal product required to sell.

---

# Post V1 Modules

## 1. PPC Intelligence

Goal: provide visibility and decision support for Amazon advertising.

Potential features:

- PPC dashboard
- ACOS / TACOS tracking
- campaign trend analysis
- alerts for unprofitable campaigns
- recommendations:
  - increase budget
  - reduce spend
  - pause campaign

Status: Not required for V1.

---

## 2. Communication Hub

Goal: centralize operational communication.

Concept:

Supplier timeline including:

- emails
- POs
- files
- notes
- activity log

Status: Post V1.

---

## 3. Internal Collaboration

Lightweight team communication.

Features:

- comments on entities:
  - project
  - purchase order
  - supplier
  - shipment
- mentions (@user)

Status: Candidate for V1.5.

---

## 4. Basic Invoicing Module

Purpose: allow sellers to generate occasional B2B invoices without leaving the platform.

Minimal scope:

- create invoice
- customer
- invoice lines
- VAT
- PDF export

Not intended to replace accounting software.

Status: Post V1.

---

## 5. Email Integration

Concept:

Send emails to suppliers directly from Freedoliapp with attachments such as:

- purchase orders
- documents
- files

Emails logged inside supplier timeline.

Status: Post V1.

---

# Strategic Rule

Ideas in this document are intentionally postponed to prevent:

- feature creep
- delayed launch
- architectural complexity before product validation.

Primary focus remains:

**Launch V1 and start generating revenue.**
