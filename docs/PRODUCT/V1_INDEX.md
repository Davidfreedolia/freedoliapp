This document defines the canonical product definition for the V1 launch.

- POSITIONING.md — Product positioning and value proposition
- STRATEGIC_POSITIONING.md — Amazon Operations OS, target user, competitors, competitive advantage, moat
- PRODUCT_LIFECYCLE_MODEL.md — Operational lifecycle (Idea → Supplier → … → Decisions), phases, UI and data implications
- PROJECT_TIMELINE_SYSTEM.md — Project-level lifecycle timeline component in Project Detail
- LIFECYCLE_EVENTS_V1.md — V1 foundation: lifecycle events (phase, PO, shipment) for decisions/automations
- DECISION_BRIDGE_LIFECYCLE_V1.md — V1 bridge: lifecycle events → decisions (phase, shipment_delivered, inventory_low_stock)
- INVENTORY_LOW_STOCK_LIFECYCLE_V1.md — V1 emitter: reorder alerts → inventory_low_stock lifecycle event (dedupe 24h)
- UI_DATA_STATES_V1.md — Standardized loading / empty / error states (DataState, DataLoading, DataEmpty, DataError)
- ENTRY_EXPERIENCE.md — Entry flow (landing → signup → activation → app)
- HOME_ARCHITECTURE.md — Home dashboard structure and widgets
- LANDING_STRUCTURE.md — Public landing page structure
- V1_EXECUTION_ROADMAP.md — Phased execution plan (Product Polish → UX Clarity → Amazon Integration → Refactor → Automation → Beta)

These documents act as the source of truth for V1.

No implementation work should contradict these documents.
